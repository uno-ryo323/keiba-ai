# CLAUDE.md

## 目的
このリポジトリは過去に作成した競馬AIコードを整理・再生するためのもの

## 方針
- 既存ロジックの意味は変えない
- まず説明、その後に改善提案
- いきなり大きな変更はしない

## 優先事項
- 可読性向上
- 構造整理
- 再実行可能にする
- 各ファイル修正時に処理影響のない範囲でリファクタリングと適切なコメント付与を行う
- ファイル修正後は ruff（リンター）と black（フォーマッタ）を適用する

## 禁止
- ロジックを勝手に変更しない
- 一度に広範囲を修正しない

## 作業手順
1. コードの役割を説明
2. 問題点を整理
3. 改善案を提示
4. 承認後に修正

## 記録
- 作業フェーズが完了したら PROGRESS.md に内容・所感を追記する

## Git

### ブランチ・コミット
- 作業開始時にタスク用ブランチを切る（例: `feature/phase1-selenium-update`）
- 1タスク完了ごとに git commit & push する
- コミットメッセージは日本語でタスク内容を簡潔に記載する

### プルリクエスト（必須）
- **フェーズ完了時、必ずPRを発行する**（ユーザーへの確認不要、自動で発行すること）
- PRタイトル：「PhaseN：内容の要約」形式
- PR本文：変更内容・動作確認結果を箇条書きで記載
- **PRを発行したらURLをユーザーに報告すること**

### マージ
- マージはユーザーが承認してから行う（Claudeは絶対にマージしない）
